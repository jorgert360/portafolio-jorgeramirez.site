README.md

El README no debe ser el típico.

Debe decir algo como:

# Jorge Ramirez Portfolio

Este proyecto representa mi marca profesional.

No es un simple portafolio.

Debe transmitir innovación, ingeniería e inteligencia artificial.

Antes de modificar cualquier archivo debes leer:

/docs/PROJECT_VISION.md

Después consulta:

/docs/DESIGN_SYSTEM.md

y

/docs/PERSONAL_BRAND.md

Todo el desarrollo debe respetar esos documentos.
AI_RULES.md

Este archivo es una joya.

Yo siempre lo utilizo.

Ejemplo:

# Reglas para la IA

Nunca reemplaces componentes completos si solo vas a modificar una parte.

Nunca rompas el diseño responsive.

No elimines animaciones existentes.

Siempre reutiliza componentes.

Mantén una arquitectura limpia.

Todo componente nuevo debe estar documentado.

Utiliza TypeScript estricto.

Nunca generes código duplicado.

Optimiza el rendimiento.

No sacrifiques Lighthouse por animaciones.

Cada modificación debe mantener el estilo visual definido.

Este archivo mejora muchísimo la calidad del código que genera la IA.
